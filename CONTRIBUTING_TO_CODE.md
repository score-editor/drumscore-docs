# Contributing Code to DrumScore Editor

Thank you for your interest in contributing to DrumScore Editor's codebase!

## About the Code Repository

DrumScore Editor's source code is maintained in a private repository. Before gaining access, contributors must sign a Contributor License Agreement (CLA).

## How to Contribute Code

### 1. Review the CLA

First, review our [Contributor License Agreement](CLA.md) to understand the terms. The CLA covers:
- Intellectual property rights
- Commercial use and revenue
- Confidentiality requirements
- Your warranties and commitments

### 2. Contact Us

If you agree to the CLA terms and want to contribute, email alan@drumscore.scot with:
- Brief introduction about yourself
- What you'd like to work on (bug fix, feature, etc.)
- Your GitHub username
- Confirmation you've read and agree to the CLA terms

### 3. Sign the CLA

We'll send you the CLA to sign digitally via OpenSign. Once signed, you'll receive access to the private code repository.

### 4. Start Contributing

After gaining access, you'll find detailed development guidelines, architecture documentation, and coding standards in the private repository's CONTRIBUTING.md file.

## What We're Looking For

Contributions in these areas are especially welcome:

- **Bug fixes**: Corrections to existing functionality
- **Drumming notation features**: Improvements specific to pipe band drumming
- **UI/UX improvements**: Better workflows for score creation
- **Performance optimizations**: Faster rendering, loading, or editing

Please discuss significant features before starting work to ensure alignment with DSE's roadmap.

## Questions?

- **Code contributions**: alan@drumscore.scot
- **Documentation contributions**: See [CONTRIBUTING.md](CONTRIBUTING.md) in this repo
- **General questions**: alan@drumscore.scot

**Note**: Documentation contributions (to this public repository) do not require the CLA - see [CONTRIBUTING.md](CONTRIBUTING.md) for documentation guidelines.
